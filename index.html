正则表达式
定义：
正则表达式(Regular Expression),是由一些普通字符和特殊字符格式组成的一串字符表达式，这个表达式用以描述一种特定的字符规则，符合这个规则的字符串就是OK的，反之不OK。

作用：
有些场景下，合格的字符串不是一个，而是一些，是一个字符串集合，这些字符串都符合某些规则。例如，我们的身份证号码校验，电话号码校验，邮箱校验，正确答案不是固定的，而是符合一定规则的字符序列。因此正则表达式常用来文本中搜索、匹配或替换特定形式的文本。

举个例子：
日常生活我们玩游戏，说名字中带有冰的人出列，那么“李冰冰”，“李冰”，“范冰冰”，“张冰”都是符合条件的答案。

初次邂逅正则
var regexp = /^1[3-9]\d{9}$/;

var input = "15513513550";
console.log(regexp.test(input));    //true

var input = "25513514920";
console.log(regexp.test(input));    //false

var input = "13313514550434324";
console.log(regexp.test(input));    //false

var input = "a1012";
console.log(regexp.test(input));    //false

正则表达式格式
/pattern/attributes
pattern: 字符串匹配规则
attribute：修饰符
修饰符 含义 示例
i ignoreCase,执行对大小写不敏感的匹配 /is/i
g global,执行全局匹配(査找所有匹配而不是在第一次匹配之后停止） /is/g

m multiline,允许执行多行匹配 /^is$/m
注：修饰符可以组合使用，如/is/igm
注：修饰符m对^和$匹配会产生影响

说明：

  • 常量字符串左右不能添加’ ‘或者” ”。例如/aaa/ 表示常量”aaa”
  • 两个/不能省略。
  • 参数pattern是一个正则表达式字符串。
  • 参数attributes是一个可选的字符串，可以在”g”、”i”和”m”范围内指定，分别用于指定g(global)全局匹配、i(ignoreCase)忽略大小写的匹配和m(multiline)多行匹配。

创建正则表达式
两种方法：
1、直接量语法：
var part1 = /pattern/attributes;
var regexp1 = /\d{2,3}/ig;
2、调用RegExp构造方法：
var part2 = new RegExp(pattern,attributes);
var regexp2 = new RegExp(‘\d{2,3}’,’ig’);
正则表达式定义
常量字符串
常量字符串就是一个普通的字符串，不含有特殊的字符。使用//将这个字符串直接包裹就构成了一个简单的正则表达式，注意前面说明提到的，正则表达式中，常量字符串写在//中间时，不加单引号或者双引号。
var regexp = /ipod/;
var data = ‘Apple iPod id No.123 cool?’
console.log(regexp.test(data)); //false
正则表达式特殊字符
字符 含义 示例
[^] 匹配不等于[]的字符 /[^a]/
^ 匹配字符串的开头 /^a/
$ 匹配字符串的结尾 /a$/
\b 匹配单词的边界 /\bhis\b/
\B 匹配单词的非边界 /\Bhis\B/
?=x 匹配其后紧接X的字符串 /do(?=not)/
?!x 匹配其后没有紧接X的字符串 /do(?!not)/

正则表达式定义字符重复
规则写法 含义 示例
n? 匹配零次或一次字符n /a?/
n 匹配零次或多次字符n /a/
n+ 匹配一次或多次字符n /a+/
n{x} 匹配字符n出现x次 /a{3}/
n{x,y} 匹配字符n出现x到y次 /a{2,4}/
n{x,} 匹配字符n的出现>=x次 /a{3,}/

n？ 例子：
var regexp = /a?bc/;
//问号表示匹配0次或者一次，所以没有a,或至少有一个a都是ok，
//也就是说只要有’bc’子串，或者‘abc’都是ok
console.log(regexp.test(‘bc’)); //true ‘bc’
console.log(regexp.test(‘ab’)); //false
console.log(regexp.test(‘aaabc’)); //true ‘abc’
console.log(regexp.test(‘aacbc’)); //true, ‘bc’

n+ 例子
var regexp = /a+bc/;
//+表示匹配1次或多次，所以必须至少有一个a才ok，
//也就是说’abc’子串，或者‘aabc’，‘aaaaabc’等是ok
console.log(regexp.test(‘bc’)); //false
console.log(regexp.test(‘ab’)); //false
console.log(regexp.test(‘aaabc’)); //true ‘aaabc’
console.log(regexp.test(‘abc’)); //true, ‘abc’

n{x, y }
var regexp = /a{2,4}bc/;
//{2,4}表示a出现2个~4个是ok,，
//也就是说只有’aabc’子串，或者‘aaabc’，‘aaaabc’是ok
console.log(regexp.test(‘bc’)); //false
console.log(regexp.test(‘abc’)); //false
console.log(regexp.test(‘aabc’)); //true ‘aabc’
console.log(regexp.test(‘aaaaabc’)); //true, ‘aaaabc’
正则表达式定义分组范围
正则表达式使用如下语法匹配一个范围内的一个字符，结合上面所讲，只要匹配到这个范围的一个字符就可以：
语法 含义 示例
[abc] 匹配指定集合内的任一字符 /[3458]/
[^abc] 匹配不在指定集合内的任意字符 /[^12679]/
[0-9] 匹配任一个数字 /[0-9]/
[a-z] 匹配任一个小写字符 /[a-z]/
[A-Z] 匹配任一个大写字符 /[A-Z]/
[A-z] 匹配大写A到小写z的所有字符，
即 A-Z[]^_’a-z /[A-z]/
exp1|exp2 使用丨进行条件选择 /ex|Ex| post | Post/
(exp1) 使用()指定分组——子表达式

[abc]
var regexp = /[abc]/;
//[abc]表示匹配上其中的一个字符即可，都没有才是false
regexp.test(“ade”); //true
regexp.test(“def”); //false
[^abc]
var regexp = /[^abc]/;
//[^abc]表示匹配上不是中括号中字符外其他一个字符即可，都没有才是false
regexp.test(“ade”); //true 匹配“d”,”e”
regexp.test(“ab”); //false

/[a-z]|8|3/
var regexp = /[aze]|3|9/;
//表示匹配上中括号的一个字符或者3，或者9，都没有才是false
regexp.test(“ade”); //true 匹配“a”,”e”
regexp.test(“d”); //false
regexp.test(“3e”); //true 匹配“3”
/[^(abc)]/
var regexp = /[^(abc)]/;
//^表示不等于后面紧跟字符，()成为一个单元，因此就是^同时作用于a,b,c
//也就是既不能匹配a,也不能匹配b,也不能匹配c
regexp.test(“ade”); //true 匹配“a”,”e”
regexp.test(“d”); //false

特殊元字符
正则表达式中可以使用如下元字符引用来进行简化
语法 含义 示例
\d 匹配一个数字 /\d/等价于/[0-9]/
\D 匹配一个非数字 /\D/等价于/[^0-9]/
\w 匹配一个数字/字母/下划线 /\w/等价于/[0-9a-zA-Z]/
\W 匹配一个非数字/字母/下划线 /\W/等价于/[^0-9a-zA-Z]/
\s 匹配一个空白字符（space） /\s/等价于/[\n\r\t\v\f]/
\S 匹配一个非空白字符 /\S/等价于/[^\n\r\t\v\f]/
. 匹配除回车和换行以外的任何单个字符 /./等价于/[^\n\r]/

总结：
正则表达式本质是定义了一个子串规则，所测试的字符串应该是大于等于这个子串的。也就是说例如，定义的规则是/abc/, 那么所测试字符串至少应该包含有’abc’这样一个substring, 否则就是不成立。
正则表达式对象属性
属性名 描述
global （只读）RegExp对象是否具有g修饰符
IgnoreCase （只读）RegExp对象是否具有i修饰符
multiline （只读）RegExp对象是否具有m修饰符
lastIndex 用于设置/获取下次匹配的起始位置
source （只读）返回模式匹配的起始位置
$1—$9 （全局属性）指代最近一次搜索中某个子表达式匹配的文本
注：没有使用g修饰符的RegExp对象不能使用lastIndex属性
注：创建的正则对象，被 括弧 括起来的部分，从左到右，依次是$1…$9。

正则表达式应用
string 对象的replace方法
第一个参数既可以是一个固定的子串，也可以是一个正则表达式对象
var regexp = /javascript/ig;
var data = ‘Write javascript should follow javaScript rules –javascript said’;
var result = data.replace(regexp,’JavaScript’);
document.write(result);

var regexp = /(red|green|blue)/ig;
        var data = 'Flowers are red,blue,yellow,or green';
        var result = data.replace(regexp,'<span style="color:$1;">$1</span>');
        document.write(result);

string对象的match方法
返回一个或多个子串/正则表达式的匹配

string对象的search方法
返回第一次出现匹配指定正则表达式子串的下标，若没有则返回-1
var regexp = /\bdo\b/ig;
var data = ‘He dose told to do’;
console.log(data.search(regexp));
//16
string对象的split方法
使用一个指定的字符串或正则表达式，对原字符串进行拆分，返回拆得的子字符串。
var regexp = /\s+/;
var data = ‘How are you ‘;
var arr = data.split(regexp);
console.log(arr);
//[‘How’,’are’,’you’,’’]

举例说明
//1.校验是否全由数字组成
/^[0-9]{1,20}$/
^ 表示打头的字符要匹配紧跟^后面的规则
$ 表示打头的字符要匹配紧靠$前面的规则
[ ] 中的内容是可选字符集
[0-9] 表示要求字符范围在0-9之间
{1,20}表示数字字符串长度合法为1到20，即为[0-9]中的字符出现次数的范围是1到20次。

//2.校验密码：只能输入6-20个字母、数字、下划线
/^(\w){6,20}$/

//3.校验用户姓名：只能输入1-30个字母组成的字符串
/^[a-zA-Z]{1, 30}$/

//4.校验URL
/^http[s]{0,1}:\/\/.+$/ 或 /^http[s]{0,1}:\/\/.{1,n}$/ (表示url串的长度为length(“https://”) + n )

//5.校验纯中文字符
/^[\u4E00-\u9FA5]+$/

[\u4E00-\u9FA5] ：是中文字符集的范围吧

封装常用正则函数
//校验手机号码：必须以数字开头，除数字外，可含有“-”
function isMobil(s)
{
var patrn=/^[+]{0,1}(\d){1,3}[ ]?([-]?((\d)|[ ]){1,12})+$/;
if (!patrn.exec(s)) return false
return true
}

//校验密码：只能输入6-20个字母、数字、下划线
function isPasswd(s)
{
var patrn=/^(\w){6,20}$/;
if (!patrn.exec(s)) return false
return true
}

//校验用户姓名：只能输入1-30个以字母开头的字串
function isTrueName(s)
{
var patrn=/^[a-zA-Z]{1,30}$/;
if (!patrn.exec(s)) return false
return true
}

//校验登录名：只能输入5-20个以字母开头、可带数字、“”、“.”的字串
function isRegisterUserName(s)
{
var patrn=/^[a-zA-Z]{1}([a-zA-Z0-9]|[.]){4,19}$/;
if (!patrn.exec(s)) return false
return true
}

//校验是否全由数字组成
function isDigit(s)
{
var patrn=/^[0-9]{1,20}$/;
if (!patrn.exec(s)) return false
return true
}
